# 요구 사항
- 고객이 식당 방문 날짜와 주문 메뉴를 입력하면 이벤트 적용 내역을 보여주는 12월 이벤트 플래너를 개발한다.
## 메뉴
```
<애피타이저>
양송이수프(6,000), 타파스(5,500), 시저샐러드(8,000)

<메인>
티본스테이크(55,000), 바비큐립(54,000), 해산물파스타(35,000), 크리스마스파스타(25,000)

<디저트>
초코케이크(15,000), 아이스크림(5,000)

<음료>
제로콜라(3,000), 레드와인(60,000), 샴페인(25,000)
```

## 이벤트

### 할인 이벤트
- 4개의 할인 이벤트가 있다.
- 크리스마스 디데이 할인을 제외한 다른 이벤트는 12.1~12.31동안 적용한다.
- 중복 할인과 증정을 허용한다.
- 총 주문 금액 10,000원 이상부터 이벤트를 적용한다.
- 음료만 주문 불가하다.
- 메뉴는 한번에 최대 20개까지만 주문 가능하다.
- 메뉴 1개는 메뉴 종류 1개가 아닌, 주문 수량 1개다.
  (e.g. 시저샐러드-1, 티본스테이크-1, 크리스마스파스타-1, 제로콜라-3, 아이스크림-1의 총개수는 7개)

<크리스마스 디데이 할인>
- 12.1~12.25일 동안 할인 금액은 1000원부터 시작해 날마다 100원씩 증가한다. (1일-1000원, 2일-1100원, ..., 25일-3400원)
- 총 주문 금액에서 해당 금액만큼 할인한다.

<평일 할인>
- 일요일~목요일 동안 디저트 메뉴를 메뉴 1개당 2023원 할인한다.

<주말 할인>
- 금요일, 토요일에 메인 메뉴를 메뉴 1개당 2023원 할인한다.

<특별 할인>
- 매주 일요일과 크리스마스 당일은 총주문 금액에서 1000원 할인한다.

### 증정 이벤트
- **할인 전 총주문 금액**이 12만원 이상이면 샴페인 1개 증정

## 배지
- **총혜택 금액**에 따라 다른 이벤트 배지를 부여한다.(※할인 금액 합계가 아니다.)
    - 5천원 이상: 별
    - 1만원 이상: 트리
    - 2만원 이상: 산타

## 12월 이벤트 플래너

### 입력
- 12월 중 식당 방문 날짜
```
26
```

- 주문 메뉴 및 개수
```
해산물파스타-2,레드와인-1,초코케이크-1
```

### 출력
- 주문메뉴
    - 주문 메뉴의 출력 순서는 에피타이저 > 메인 >디저트 > 음료 순으로 한다.
- 할인 전 총주문 금액
- 증정 메뉴
    - 증정 이벤트에 해당하지 않으면 `없음`을 출력한다.
- 혜택 내역
    - 적용된 이벤트 내역만 출력한다.
    - 적용된 이벤트가 하나도 없다면 `없음`을 출력한다.
    - 이벤트 출력 순서는 크리스마스 디데이 할인 > 평일 할인 > 주말 할인 > 특별 할인 > 증정 이벤트 순으로 한다.
- 총혜택 금액<br>
  `총혜택 금액 = 할인 금액의 합계 + 증정 메뉴의 가격`
- 할인 후 예상 결제 금액<br>
  `할인 후 예상 결제 금액 = 할인 전 총주문 금액 - 할인 금액의 합계`
  ※증정 메뉴의 가격은 할인 금액에 포함되지 않는다.
- 12월 이벤트 배지
    - 이벤트 배지가 부여되지 않는 경우 `없음`을 출력한다.

#### 출력 예시
```
안녕하세요! 우테코 식당 12월 이벤트 플래너입니다.
12월 중 식당 예상 방문 날짜는 언제인가요? (숫자만 입력해 주세요!)
3
주문하실 메뉴를 메뉴와 개수를 알려 주세요. (e.g. 해산물파스타-2,레드와인-1,초코케이크-1)
티본스테이크-1,바비큐립-1,초코케이크-2,제로콜라-1
12월 3일에 우테코 식당에서 받을 이벤트 혜택 미리 보기!
 
<주문 메뉴>
티본스테이크 1개
바비큐립 1개
초코케이크 2개
제로콜라 1개
 
<할인 전 총주문 금액>
142,000원
 
<증정 메뉴>
샴페인 1개
 
<혜택 내역>
크리스마스 디데이 할인: -1,200원
평일 할인: -4,046원
특별 할인: -1,000원
증정 이벤트: -25,000원
 
<총혜택 금액>
-31,246원
 
<할인 후 예상 결제 금액>
135,754원
 
<12월 이벤트 배지>
산타
```

### 유효성 검증

<입력 값 검증 시 공통 사항>
- 유효한 입력 값이 아니면 에러 메시지를 출력하고 재입력 받는다.
- 입력 값이 공백이거나 빈 값일 경우<br>
  `[ERROR] 공백이 아닌 문자를 입력해주세요.`

<방문 날짜 입력 시>
- 숫자(정수)가 아닐 경우<br>
  `[ERROR] 숫자가 아닙니다. 다시 입력해주세요.`
- 1 이상 31 이하의 숫자가 아닐 경우<br>
  `[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요.`

<주문 메뉴 및 개수 입력 시>
- 입력 형식에 맞지 않을 경우<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 메뉴가 메뉴판에 없는 경우<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 메뉴가 음료 뿐일 경우<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 메뉴가 중복될 경우(e.g. 시저샐러드-1,시저샐러드-1)<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 각 메뉴의 개수가 1이상이 아닐 경우<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`
- 메뉴의 개수가 20개를 초과할 경우<br>
  `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.`

### 기능 목록
1. 방문 날짜 입력 안내 문구를 출력한다.
2. 방문 날짜를 입력 받는다.
   2-1. 입력 받은 방문 날짜를 검증한다.
   2-2. 유효하지 않은 값일 경우 에러 메시지를 출력하고 재입력 받는다.
3. 주문 메뉴와 개수를 입력 받는다.
   3-1. 입력 받은 메뉴와 개수를 검증한다.
   3-2. 유효하지 않은 값일 경우 에러 메시지를 출력하고 재입력 받는다.
4. 영수증에 입력 받은 주문 메뉴와 개수를 담는다.
5. 메뉴와 영수증을 비교해 할인 전 총주문 금액을 계산한다.
6. 영수증의 주문 메뉴와 개수, 할인 전 총주문 금액을 출력한다.
7. 증정 이벤트에서 영주증의 총주문 금액을 비교해 증정 메뉴를 출력한다.
   7-1. 총 주문 금액이 12만원 이상이면 증정 메뉴에 "샴페인"을 출력한다.
   7-2. 총 주문 금액이 12만원 미만이면 증정 메뉴에 "없음"을 출력한다.
8. 할인 이벤트들 중 방문 날짜에 적용되는 할인 이벤트를 혜택내역으로 가져온다.
9. 혜택 내역에서 8의 할인 이벤트와 영수증을 비교해 할인 이벤트별 할인 금액을 계산한다.
10. 혜택 내역에서 증정 이벤트와 총주문 금액을 비교해 증정 이벤트의 할인 금액을 계산한다.
11. 혜택 내역을 출력한다.
12. 영수증에서 혜택 내역을 가져와 총혜택 금액을 계산한다.
13. 총혜택 금액을 출력한다.
14. 영수증에서 할인 전 총주문 금액과 13에서 가져온 혜택 내역 중 할인 금액의 합계로 할인 후 예상 결제 금액을 계산한다.
15. 할인 후 예상 결제 금액을 출력한다.
16. 이벤트 배지에서 영수증의 총혜택 금액을 가져와 이벤트 배지를 적용한다.
17. 이벤트 배지를 출력한다.
    17-1. 적용되는 이벤트 베지가 없으면 "없음"을 출력한다.

#### 객체들의 역할
<메뉴>
- 식당의 모든 메뉴와 가격을 Enum으로 관리한다.

<영수증>
- 주문 내역을 받아 저장한다.
- <메뉴>를 받아 총주문 금액을 반환한다.
- 혜택 목록을 가져와 총혜택 금액을 반환한다.
- 가져온 혜택 목록 중 할인 금액의 합계를 반환한다.
- 할인 후 예상 결제 금액을 반환한다.

<혜택 내역>
- 혜택 목록을 저장한다.
    - 혜택 목록은 각 이벤트별 할인 금액을 가진다.
- 할인 이벤트들 중 입력받은 방문날짜에 적용되는 할인 이벤트를 가져와 혜택 목록에 추가한다.
- 영수증의 총주문 금액과 증정 이벤트를 비교해 혜택 목록에 추가한다.
- 영수증의 주문 내역을 가져와 할인 이벤트별 할인 금액을 계산한다.

<이벤트 배지>
- 배지를 관리한다.
- 영수증의 총 혜택 금액을 가져와 적절한 이벤트 배지를 반환한다.

<이벤트>
-  <할인 이벤트>와 <증정 이벤트>의 상위 클래스다.
- 이벤트를 적용해 금액을 반환하는 추상 메서드를 가진다.

<할인 이벤트>
- <이벤트>를 상속받는다.
- <크리스마스 디데이 할인>, <평일 할인>, <주말 할인>, <특별 할인>의 상위 클래스다.
- 날짜와 영수증을 받아 할인 금액을 반환하는 추상메서드를 가진다.

<크리스마스 디데이 할인>
- <할인 이벤트>를 상속받는다.
- 날짜가 영수증을 받아 할인 금액을 반환한다

<평일 할인>
- <할인 이벤트>를 상속받는다.
- 날짜와 영수증을 받아 할인 금액을 반환한다.

<주말 할인>
- <할인 이벤트>를 상속받는다.
- 날짜와 영수증을 받아 할인 금액을 반환한다.

<특별 할인>
- <할인 이벤트>를 상속받는다.
- 날짜와 영수증을 받아 할인 금액을 반환한다.

<증정 이벤트>
- <이벤트>를 상속받는다.
- 총 주문 금액을 받아 증정 메뉴의 금액을 반환한다.

<이벤트 배지>
- 배지와 적용 기준 금액을 Enum으로 관리한다.
- 영수증의 총혜택 금액을 받아 적용된 배지를 반환한다.
